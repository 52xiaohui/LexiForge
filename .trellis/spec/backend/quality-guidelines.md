## Required Patterns

<!-- Patterns that must always be used -->

### AI Article Response Normalization

AI-generated article JSON is an external boundary, even when `response_format`
uses `json_schema` with `strict: true`.

#### 1. Scope / Trigger
- Trigger: backend code consumes article content generated by an
  OpenAI-compatible model.
- Applies to: `backend/internal/ai` response parsing and
  `backend/internal/article` coverage validation.

#### 2. Signatures
- `ai.Client.GenerateArticle(ctx, req) (*ai.GenerateArticleResponse, error)`
- `article.locateCoverage(content, targets, coveredWords) ([]ArticleWordDraft, int)`

#### 3. Contracts
- The AI client should request strict schema output, but must tolerate common
  provider/model drift at the parsing boundary.
- `content_markdown` is required. Empty content is an invalid AI response.
- Missing `title`, `summary`, or `missing_words` may be repaired with safe
  local defaults.
- `covered_words[].word` may be accepted as an alias for `spelling`.
- `covered_words[].occurrence` may be accepted as either an integer or a
  numeric string.
- Article coverage must be based on actual `content_markdown`, not only the
  model's `covered_words` claims.

#### 4. Validation & Error Matrix
- Empty chat content -> `ErrInvalidAIResponse`.
- Unparseable JSON object -> `ErrInvalidAIResponse`.
- Empty `content_markdown` -> `ErrInvalidAIResponse`.
- Invalid `covered_words` claim context -> ignore the claim and verify by
  searching the article text.
- Target word appears as a simple plural in text, such as `pound` -> `pounds`
  -> count it as covered when it is a whole word.
- Target spelling only appears as a substring, such as `fund` in `fundraising`
  -> do not count it as covered.

#### 5. Good/Base/Bad Cases
- Good: fenced JSON with `word` instead of `spelling`, missing optional fields,
  and `"occurrence":"1"` parses to a usable article response.
- Base: strict schema JSON parses without modification.
- Bad: trusting `covered_words` without verifying adjacent context can store
  false coverage.

#### 6. Tests Required
- AI client tests must cover schema-drift normalization at the HTTP response
  boundary.
- Article service tests must cover claim verification, text fallback, word
  boundary protection, and simple plural coverage.

#### 7. Wrong vs Correct

Wrong:

```go
// Trusts the model's coverage claim without checking article text.
covered := len(aiResp.CoveredWords)
```

Correct:

```go
words, covered := locateCoverage(aiResp.ContentMarkdown, targets, aiResp.CoveredWords)
```

## Testing Requirements

<!-- What level of testing is expected -->

- Bug fixes for AI response instability require regression tests for both the
  model boundary (`backend/internal/ai`) and the article coverage validator
  (`backend/internal/article`) when both layers are involved.
